# Model Notes

## Generation model

LaunchBot uses:

```bash
ollama pull llama3.2
```

The `.env` value is:

```bash
GENERATION_MODEL=llama3.2
```

This aligns with earlier curriculum work that used Llama 3.2 through Ollama. It may be too slow on `t3.micro`, so upgrade to `t3.medium` if the chatbot times out.

## Embedding model

LaunchBot uses:

```bash
ollama pull nomic-embed-text
```

The `.env` value is:

```bash
EMBEDDING_MODEL=nomic-embed-text
```

This is a dedicated embedding model used to create vectors for Chroma retrieval.

## What if I used embeddinggemma before?

You may have used `embeddinggemma` earlier in the program. That is also an embedding model, but this deployable app uses `nomic-embed-text` by default to keep the deployment instructions consistent and predictable.

If you change the embedding model, delete and reseed Chroma:

```bash
cd /var/www/launchbot/server
source .venv/bin/activate

rm -rf instance/chroma
python seed_chroma.py
```

Do this because vectors stored with one embedding model may not be compatible with vectors generated by another embedding model.
